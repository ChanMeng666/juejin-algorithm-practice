# Ad Title Wildcard Matching Problem

## Problem Description

In an advertising platform, to improve the efficiency and flexibility of creating ad titles, the system allows submitting creatives using wildcards. Wildcards are enclosed in paired curly braces `{}` and can contain 0 or more characters. The system needs to determine whether a given title can be generated by replacing the wildcards in the template.

### Requirements
- `n`: Number of titles to evaluate
- `template`: Template string containing wildcards
- `titles`: List of titles to evaluate

### Constraints
Return a string containing n comma-separated True or False values, indicating whether each title can be generated from the template.

## Examples

**Example 1:**
```python
Input:
n = 4
template = "ad{xyz}cdc{y}f{x}e"
titles = ["adcdcefdfeffe", "adcdcefdfeff", "dcdcefdfeffe", "adcdcfe"]
Output: "True,False,False,True"
```

**Example 2:**
```python
Input:
n = 3
template = "a{bdc}efg"
titles = ["abcdefg", "abefg", "efg"]
Output: "True,True,False"
```

## Solution Approach

This is a string matching problem that can be solved using dynamic programming. The main steps are:

1. **Template Parsing**:
   - Parse the template string into fixed parts and wildcard parts
   - Wildcards can match strings of any length (including empty strings)

2. **Dynamic Programming**:
   - Create a dp array: `dp[i][j]` indicates whether the first i parts of the template can match the first j characters of the title
   - For fixed string parts, exact matching is required
   - For wildcard parts, any length of string can be matched

3. **Result Processing**:
   - Perform matching for each title
   - Combine results into the required format

## Implementation

```python
def solution(n, template_, titles):
    def can_match(template, title):
        # Split the template by {}
        parts = []
        i = 0
        fixed_parts = []
        while i < len(template):
            if template[i] == '{':
                start = i
                while i < len(template) and template[i] != '}':
                    i += 1
                if i < len(template):
                    parts.append(('wildcard', start))
                i += 1
            else:
                start = i
                while i < len(template) and template[i] != '{':
                    i += 1
                fixed_parts.append(template[start:i])
                parts.append(('fixed', len(fixed_parts) - 1))
        
        # Dynamic programming array
        dp = [[False] * (len(title) + 1) for _ in range(len(parts) + 1)]
        dp[0][0] = True
        
        # Fill the dp array
        for i in range(1, len(parts) + 1):
            part_type, part_index = parts[i-1]
            
            if part_type == 'wildcard':
                # Wildcard can match any characters
                for j in range(len(title) + 1):
                    if dp[i-1][j]:
                        for k in range(j, len(title) + 1):
                            dp[i][k] = True
            else:
                # Fixed string must match exactly
                fixed_str = fixed_parts[part_index]
                for j in range(len(title)):
                    if dp[i-1][j]:
                        if j + len(fixed_str) <= len(title):
                            if title[j:j+len(fixed_str)] == fixed_str:
                                dp[i][j+len(fixed_str)] = True
        
        return dp[len(parts)][len(title)]

    # Process each title
    results = []
    for title in titles:
        results.append(str(can_match(template_, title)))
    
    return ','.join(results)
```

## Code Explanation

1. **Template Parsing**:
   ```python
   while i < len(template):
       if template[i] == '{':
           # Handle wildcard part
           parts.append(('wildcard', start))
       else:
           # Handle fixed string part
           fixed_parts.append(template[start:i])
           parts.append(('fixed', len(fixed_parts) - 1))
   ```
   - Decompose the template into two types of parts: fixed strings and wildcards
   - Use tuples `(type, index)` to record the type and position of each part

2. **Dynamic Programming**:
   ```python
   dp = [[False] * (len(title) + 1) for _ in range(len(parts) + 1)]
   dp[0][0] = True
   ```
   - Create the dp array with the initial state set to True
   - dp[i][j] indicates whether the first i parts of the template can match the first j characters of the title

3. **Match Processing**:
   - For wildcards: can match strings of any length
   - For fixed strings: must match exactly

## Complexity Analysis

- Time complexity: O(n * m * k)
  - n: template length
  - m: title length
  - k: number of wildcards
- Space complexity: O(m * p)
  - m: title length
  - p: number of template parts

## Summary

This problem is a classic string matching problem solved using dynamic programming. The key points are:
1. Correctly parsing the template string
2. Using dynamic programming to handle different types of matching cases
3. Flexibly handling wildcard matching rules
